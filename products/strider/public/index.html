Strider CD

Open Source Continuous Integration & Deployment Server

Build and test code continuously

Continuous integration and deployment increases productivity and reliablity by ensuring your code is always tested, and built automatically. This is just the beginning. Tests are passing? Strider enables you to automate any deployment process, no matter how complex. Whether you use Ansible, Chef, Puppet, Heroku, SSH or straight Shell, Strider's got your back.

Integrate seamlessly with GitHub

Simply add your Github projects to Strider, and relax as the tight integration means your tests will run on every commit. Using something else? Our community has contributed plugins for all the most popular repository hosts.

Fully compatible with Docker

Strider was built before Docker, but Strider's modular architecture meant utilizing Docker was no problem. We're now 100% compatible with Docker. Docker Runner allows you to test your code in isolation within a Docker container. The Docker Build plugin allows you to build images! Combine both for huge gains in automation.

Modular & Extensible

Every component in Strider is organized as a separate NPM package. Strider plugins are also just NPM packages. The Strider project composes these packages to provide you with an initial distribution that works right out of the box and can also be easily extended as needed. You can generate a new plugin anytime by running `strider init`
